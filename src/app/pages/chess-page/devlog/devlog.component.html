Switch Colour, Flip Board, Pins, Blocks and Refactors 17th-19th January 2023
  • Added the ability to switch the board to play as either Black or White
  • Started work on developing a legal move generator from the psudomove generator that accounts for the legal moves involved when the king is under check or pieces being pinned by sliding pieces and the ability to block checks by pushing pieces into the path of attacking pieces.
  • Refactored the pseudolegal move generator to reduce duplicate coded.
    • Websocket Upheaval 14th-16th January 2023
  • At this point the game was running noticeably slowly waiting for the API requests to send and authenticate requests. It was time to switch to a different method of server communication.
  • Multiplayer games and chatboxes use Websockets to communicate, here a connection is made between the server and client that messages can be sent both ways, the connection is only closed once the game has concluded so this means we can get a quick back and forth between the server and client for much smoother gameplay and also the ability for the server to tell the second player when a move has been made instead of making constant requests until a change has happened to the game state.
  • Spring has the capability to create these websocket connections and JavaScript librarys Stomp and sockJS can handle the client side.
    • Legal Moves 11th-13th January 2023
  • Worked on a move generator that creates all legal moves for each piece.
  • Added special move for castling, and defenders for each square so that the king cannot move to a square under attack.
  • Used perft testing to validate the accuracy of the move generator, making adjustments as needed.
    • Pseudo-Legal Moves 9th-10th January 2023
  • Began work on the backend game mechanics engine.
  • Wrote code to handle the movement of pieces and implement the rules of chess.
  • At this stage, all moves are pseudo-legal, meaning they are technically valid moves, but they may not necessarily be legal within the context of the current game state (e.g. a move that puts one's own king in check).
    • Moving Pieces 8th January 2023
  • Added the ability to move the pieces on the board.
  • Snap to grid was implemented and the moves onto a square were sent through a API request to the Spring server.
